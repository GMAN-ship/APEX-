Ghost-Master Terminal

Complete Production Blueprint: AI Prompts & Native Android APK Export Guide

To turn your Lovable / Supabase / MT5 Trading Terminal into a tangible, downloadable Android .apk file that runs perfectly on any phone, you need a two-tier approach.

Architecture Concept: Your application behaves as a production-grade interface mapped into a native Android wrapper engine.

Two-Tier Approach

  1. Tier 1 - Web Application: React/Vite web app with Supabase real-time backend and MT5 integration
  2. Tier 2 - Native Wrapper: Capacitor bridges your web app into native Android APK

Build Process

cd webapp npm install npm run build npm install @capacitor/core @capacitor/cli npx cap init "APEX EA" com.apex.ea --web-dir=dist npx cap add android npx cap copy android cd android ./gradlew assembleDebug

Automated GitHub Actions

Your project now includes a complete CI/CD pipeline that:

  • Automatically builds APK on every push to main/develop
  • Uploads artifacts to GitHub Actions for download
  • Creates releases when you tag a commit
  • Signs and optimizes the APK for distribution

⚠️ Important: Each build produces a .apk file ready for installation on Android devices. No PDF export needed—your app IS the product.

Installation on Device

# Connect device via USB with Developer Mode enabled adb install app-debug.apk # Or scan QR code from GitHub release
